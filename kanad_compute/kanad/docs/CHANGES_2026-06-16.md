# Kanad changes — 2026-06-16 (solver-protocol hardening + rigorous-audit bug fixes)

Dev-facing changelog for **kanad-app** and **kanad-compute** maintainers. A rigorous
test-audit (855 tests passed but had *facade-only* coverage) surfaced 17 HIGH + ~23 medium/low
bugs in untested paths. The HIGH bugs are now fixed and regression-tested. Full inventory:
`kanad/AUDIT_WORKLIST.md`. Tier-1–3 plan: `kanad/FIX_AND_IMPROVE_PLAN.md`.

## ⚠️ Behavioral changes downstream code MUST adapt to

1. **`HybridSubspaceVQE` is RETIRED** (removed from `kanad.solvers.__all__`). It was broken
   (crashed on every input — read `.parameters` off a frozen `SolverResult`) and non-conformant.
   **Action:** replace any `HybridSubspaceVQE` use with `SamplingSQDSolver`.
2. **`ConfigurationExplorer` now returns the *correlated* energy** (per `solver_type`) for PES /
   barrier scans, not the HF energy it silently returned before. PES curves / barriers will change
   (they are now correct). `solver_type='hf'` keeps the cheap HF path; unknown values now raise.
3. **`Bond.compute_energy(method='vqe'|'sqd')` now returns a finite energy** instead of crashing
   (it subscripted a frozen `SolverResult`). Both covalent and ionic bonds fixed; `subspace_size`
   is now populated correctly.
4. **`DeterministicCI` / `SQDSolver` is now correct on multireference systems.** It previously
   returned an above-FCI energy with `converged=True` (wrong spin sector). Energies on stretched /
   strongly-correlated systems will change (now exact FCI in the active space).
5. **Excited states on `mp2no` / `avas` active spaces are now correct.** MRPT (`_solve_ci_mrpt`) and
   SA-CASSCF (`_excited_states_sa_casscf`) used canonical MOs instead of the rotated active-space
   MOs; excited-state energies for those strategies will change (now variational/correct).
6. **`compute_quantum_dos(solver='sqd')` now uses the full SQD spectrum** (it discarded it before).
7. **Cloud backends (`ibm`/`bluequbit`/`ionq`) are now `BaseBackend`-conformant** (name + protocol
   methods), so `solver.backend_name` and the backend protocol work for them.
8. **`BondFactory.create_bond(Atom, Atom, distance=...)`** now honors `distance` (it was ignored for
   the Atom+Atom path — only the string path repositioned).
9. **`VQESolver(..., ansatz=<obj>)` now uses the passed ansatz** in bond mode (it was overwritten).
10. **Reactions:** legacy `ReactionSimulator`/`QuantumReactionSimulator` quantum energies were
    computed at the wrong geometry (`unit='bohr'` on Angstrom positions) — fixed; a failed quantum
    solve no longer silently substitutes a classical Morse/LJ energy (it raises); IRC barriers no
    longer come out negative (backward branch was climbing uphill).

- **HIGH (this audit):** H1 ConfigExplorer-HF, H3/H4 MRPT/SA-CASSCF rotated-MO, H5 Bond
  SolverResult subscript, H6 DOS SQD spectrum, H7 UVVis/vibronic crash, H8 cloud-backend protocol,
  H9 BondFactory distance, H12 HybridSubspaceVQE retire, H13 IRC negative barriers, H14
  `diagonalize_pyscf` spin_s OOM guard, H15 ActiveHamiltonian protocol, H16 VQE ansatz, H17 LUCJ
  adjoint-gradient flag, plus DeterministicCI multireference, reaction Morse-fallback + Angstrom/Bohr.
- **Tier 1–3 (earlier):** `decompose_energy` on active spaces, `BondingAnalyzer` gap for builder
  systems, `mp2no` near-single-reference fallback, FCI `max_memory` sizing, `DOSCalculator` molecular
  Hamiltonian, vacuous-SQD-ansatz QPU guard, minimal-basis reactivity warning. (`ci_backend='custom'`
  verified == FCI; several "open" items verified already-correct.)

## Known / deferred (NOT silent-wrong)

- Paramagnetic NMR not implemented (diamagnetic-only, **flagged** via `paramagnetic_part_missing=True`).
